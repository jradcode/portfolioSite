About Me

I was born and raised in San Jose, CA. Growing up in a working-class neighborhood, I was introduced to computers at a very young age. I used the old Apple IIe/c before there was even a mouse. I used to play the game Oregon Trail in class in elementary school. I also used the old black and gray Apple Macintosh 128K when it first introduced the concept of a mouse as a point-and-click user interface. We ended up getting an old one for home and used it for writing papers for school and getting used to the operating system and old applications. It had a GUI interface unlike the green command-line driven computers.

In middle school, I learned and worked with the old Windows 95 Zenith computers. I remember first using a chatbot called Conversation that was the introduction of the new AI LLMs of today. It was a simple chat-like program that helped people improve their typing skills by asking and answering open-ended questions. It was fun messing around with it. I also worked in the computer lab helping turn on and shut down computers after use.

For college, I went to Evergreen Valley Community College and studied Computer Information Technology (CIT). CIT focused around learning complex systems and overall computer technology. Instead of math like computer science, it's focused on the meat and potatoes of computers. I learned programming, relational database design, web development, and any new computer technologies at the time. The first programming language I learned was C++ using Dev C++ as the compiler. I mostly learned basic programming structures and logic and did problems out of the textbook. The problems were typical algebra math-type problems and simple command-line applications like accounting apps and payroll-type apps. It was a strong introduction to programming as C++ has more programming concepts to learn than simple scripting languages.

Next, I learned web publishing with HTML and XHTML, as at the time they were competitors before the rise of HTML5. Along with that, I learned CSS, as you need to learn that stuff to style web pages. The final class project was to create a full front-end website (no back-end). I created a simple political website with images and text with all the knowledge I learned.

Later on, I learned Perl (Perl 5) which back then was a vital technology to create the back-end of a website to transfer data from the front-end to the server side and used the DB and CGI modules. Perl was much more relevant back then than now, as it was once the "glue" of the internet. Now it's all replaced by Python, Ruby, Java, PHP, and other modern languages and frameworks. With Perl, I created in class a simple CGI script and a database interface for a simple site. Perl's main superpower is text manipulation with a strong Regular Expression engine that is still one of the best by today's standards.

Along with HTML, I also learned JavaScript in another college class. I learned the DOM (Document Object Model) and created small web apps from existing HTML pages. Like for example, creating a clock with images, or building an image carousel or adding animation effects on a site to enhance it. Back then there were few frameworks and I had to create them from scratch.

Back then Java and Object-Oriented Programming were popular, and I took a class to learn that as well. It was actually one of the hardest languages to learn as you have to learn a new programming design structure rather than a straightforward procedural structure. I felt like you had to think in the future for the design of the app. With Java, I created a simple calculator and simple command-line POS system.

Later on, I started learning new programming languages myself to keep up with the times. I worked and went to UC Santa Cruz and worked "survival jobs." I learned many different programming languages like Python (2 and 3), Django/Flask, Ruby/Ruby on Rails, Sinatra, PHP, Laravel, Node.js, Express/Angular/React/Vue, and more recently learning C# ASP.NET, .NET Core/Blazor frameworks throughout my life. I did a little of Windows Forms too. I'm more of a generalist as it's easier to navigate to different frameworks and languages. I'm more on and off in my IT learning as I learned other things, but always ended up going back to programming technologies.

My interests are computers, and I love video games. I like different genres of games from FPS to 4X strategy games. I like to learn and experiment with new things. In terms of operating systems, I prefer Windows as I use it to play games and code in C#. But I have also learned much of Linux and installed many other distros. Overall, I'd say my biggest focus is in web apps and full-stack development as I learn how the whole web project is put together.
